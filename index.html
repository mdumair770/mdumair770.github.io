Project Documentation
Automated Cloud Infrastructure Audit System
About This Project

Cloud computing is everywhere — but most cloud projects you see in student portfolios look the same. Spin up an EC2 instance, host a static website on S3, maybe build a serverless function with Lambda. These are useful exercises, but they do not reflect what cloud teams actually spend their time doing.

The reality is that most enterprise cloud work is about governing, securing, monitoring, and auditing infrastructure that already exists. This project answers all of those questions — automatically. A complete cloud governance system on AWS that continuously monitors an entire account for security violations, compliance failures, and cost anomalies.

This is not a tutorial project. This is the kind of infrastructure that companies pay consultants thousands of dollars to set up.

Key Audit Findings
🔴 CRITICAL — Unrestricted Security Groups
9 of 9 security group rules allow 0.0.0.0/0 inbound access — any IP on the internet can attempt connection to these ports.
🔴 HIGH — CloudTrail Not Enabled
Account activity logging is inactive — no audit trail exists for any user or API action in the account.
🔴 HIGH — GuardDuty Not Active
AI-powered threat detection is not running — the account has no automated threat monitoring.
🟡 MEDIUM — IAM MFA Gaps
MFA is not enforced for all IAM users and console access — credential theft risk is elevated.
Conclusion

Cloud governance is a core requirement for any organization running workloads on AWS. This project demonstrates a practical, working solution covering security, compliance, and cost governance simultaneously. The audit system is live, continuously running, and already producing real findings.

The technology is real. The findings are real. The system is real.
