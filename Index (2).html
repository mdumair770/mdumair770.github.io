About This Project

Cloud computing is everywhere — but most cloud projects you see in student portfolios look the same. Spin up an EC2 instance, host a static website on S3, maybe build a serverless function with Lambda. These are useful exercises, but they do not reflect what cloud teams actually spend their time doing in the real world.

The reality is that most enterprise cloud work is not about building applications. It is about governing, securing, monitoring, and auditing the infrastructure that already exists. Cloud Security Engineers, FinOps Analysts, Cloud Architects, and Compliance teams spend their days asking: Is our environment secure? Are we following the rules? Are our costs under control? Who changed what and when?

This project answers all of those questions — automatically. I designed and deployed a complete cloud governance system on AWS that continuously monitors an entire account for security violations, compliance failures, and cost anomalies, and delivers real-time alerts the moment something goes wrong. This is not a tutorial project. This is the kind of infrastructure that companies pay consultants thousands of dollars to set up.

Key Audit Findings

🔴 CRITICAL — Unrestricted Security Groups
9 of 9 security group rules allow 0.0.0.0/0 inbound access — any IP on the internet can attempt connection to these ports.
🔴 HIGH — CloudTrail Not Enabled
Account activity logging is inactive — no audit trail exists for any user or API action in the account.
🔴 HIGH — GuardDuty Not Active
AI-powered threat detection is not running — the account has no automated threat monitoring.
🟡 MEDIUM — IAM MFA Gaps
MFA is not enforced for all IAM users and console access — credential theft risk is elevated.

Conclusion

Cloud governance is not a niche skill — it is a core requirement for any organization running workloads on AWS. Security misconfigurations, compliance gaps, and uncontrolled spending are among the most common and most costly problems cloud teams deal with every day.

This project demonstrates a practical, working solution to all three. The audit system is live, continuously running, and already producing real findings. Every component was chosen deliberately, configured intentionally, and connected to deliver a complete end-to-end governance framework.

The technology is real. The findings are real. The system is real.
